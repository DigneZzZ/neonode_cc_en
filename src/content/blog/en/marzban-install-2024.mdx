# My Philosophy for Choosing a Rental Server

## Philosophy

If your Marzban server is already generating or planning to generate significant income, you need to ensure a sufficient level of reliability for your infrastructure.
1. Don‚Äôt skimp on high-quality and reliable servers.
2. Don‚Äôt skimp on resources for good load handling.
3. Don‚Äôt skimp on the cost of your nodes ‚Äì connection stability and quality are essential for the growth of your "business"/"project."
4. Ideally, get main servers from major providers focused on corporate clients. Examples: Selectel, Cloud.Vk.Com, Yandex Cloud, Cloud.ru, etc.
You may also consider providers like AWS, Vultr, Kamatera, Hetzner, but again ‚Äì choose plans with dedicated cores, not shared, and ensure 100% guarantee that your server won‚Äôt be deleted and you can always pay for it.
5. Situations with sites can vary widely, from DDoS attacks to client influx. But, of course, don‚Äôt forget about economics.
6. The better the provider and channels, the happier your clients will be. Don‚Äôt strive to offer 30 countries to choose from ‚Äì your clients won‚Äôt appreciate it. Base your choice on your client‚Äôs geography. For Russia, a server in Europe is ideal. Very rarely will anyone need a location in the US‚Ä¶ let alone somewhere like Chile or Australia.

> Thus, the provider should be stable, and ideally foreign.

I have already shared my recommendations in the VPS Insider channel (though, due to lack of interest, I stopped updating it, leaving the full history there) ‚Äì you can join via the link for 170 TG Stars: https://t.me/+lQBQTGym57hmOTAy

- Speed should be 10Gbps ‚Äì at least 2-5Gbps. 
- Traffic from 3-5TB (total). Of course, unlimited is better, but quality servers usually have limited traffic. The final choice will depend on your total traffic.

> Having a 10Gbps server but with 1 core and 1GB of memory ‚Äì you will never get maximum speed. Already with a stable load of 1Gbps, your 1 core will run at 100%, and you‚Äôll

 lose traffic, speed, and clients.

# Installation

At the time of writing, my PR for installation scripts is under review: [Marzban PR](https://github.com/Gozargah/Marzban-scripts/pull/29)

Installation script:

```bash
sudo bash -c "$(curl -sL https://github.com/DigneZzZ/Marzban-scripts-beta/raw/main/marzban.sh)" @ install --database mariadb
```

You will end up with Marzban installed but without a domain.

## Create admin 

```bash
sudo marzban cli admin create --sudo
```

## Domain Binding in CloudFlare

Once you have chosen and purchased a domain, you need to host it somewhere and bind it to the server.

CloudFlare is the best option for this.

## Certificate Issuance

For these tasks, Acme.sh is an excellent solution.

Generate it on your server for your domain:

```bash
~/.acme.sh/acme.sh --set-default-ca --server letsencrypt --issue --standalone \
-d DOMAIN \
--key-file /var/lib/marzban/certs/key.pem \
--fullchain-file /var/lib/marzban/certs/fullchain.pem
```

Replace **DOMAIN** with your purchased and bound domain to your server.

## Adjust the .env File

### Open the .env file with the command:

```bash
marzban edit-env
```

### Change the port to 443
```bash
UVICORN_PORT = 443
```

### Add records to the obtained certificates

```shell
UVICORN_SSL_CERTFILE = "/var/lib/marzban/certs/fullchain.pem"
UVICORN_SSL_KEYFILE = "/var/lib/marzban/certs/key.pem"
```

That‚Äôs basically it.

Now, all that remains is to add the configuration to Xray and start using it.
